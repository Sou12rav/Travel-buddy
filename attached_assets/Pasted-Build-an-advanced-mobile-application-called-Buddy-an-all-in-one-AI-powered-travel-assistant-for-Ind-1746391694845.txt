Build an advanced mobile application called Buddy, an all-in-one AI-powered travel assistant for India. The app should combine travel planning, live updates, chatbot support, location-aware suggestions, bookings, and visual guidance. The centerpiece is Buddy, an AI chatbot like ChatGPT, capable of handling any travel-related (or casual) query with updated, conversational, and context-aware responses.

🤖 Chatbot: Buddy (like ChatGPT, but for Travelers)
Tech Base: GPT-based language model with real-time function-calling and API integration.

Tone: Friendly, warm, informative — like a well-traveled friend who knows everything.

Memory: Should retain conversation context for follow-up queries.

Response Quality: Human-like, accurate, fast, updated.

🗺️ Map & Real-Time Interface
Interactive map interface that:

Highlights temperature, hotels, food joints, and sightseeing spots.

Marks safety zones (color-coded), and "traveler alerts."

Works by detecting user’s location OR selecting a city (e.g., "Kolkata").

Shows nearby:

Hospitals

Hotels

ATMs

Bookable cabs or autos

Tourist attractions

Location-aware chatbot support: “What’s cool near me?”

✨ Design Brief (Elegant & Attractive UI)
Theme: Smooth gradients, modern icons, immersive city visuals.

Interface:

Home screen with current location, weather, alerts, and “Ask Buddy” button.

Chatbot screen like ChatGPT with voice input, memory bubbles, and clickable suggestions.

Maps panel for destination-based view.

Experience: Should feel premium, intuitive, and built for passionate travelers.

🔧 Suggested Tech Stack
Frontend: Flutter or React Native

Backend: FastAPI + Python or Node.js

Chatbot: OpenAI GPT with tools/functions for:

Weather (OpenWeather)

Maps/Places (Google Maps, Mapbox)

News (NewsAPI)

Booking (Zomato, Uber/Ola, Booking.com, MakeMyTrip APIs)

Emergency contact (Indian public datasets)

Database: Firebase / MongoDB

Hosting: AWS / Google Cloud
